## Two Pointer Pattern
The core idea is simple: instead of using nested loops (O(n²)), 
you place two pointers at different positions in the array and move them smartly 
— reducing it to O(n).

## The Mental Model
Think of it like two people walking through a sorted array from opposite ends, 
closing in on each other until they meet.

[ 1,  2,  3,  4,  5,  6,  7 ]
  ↑                        ↑
 left                    right


At each step, you make a decision — move left forward, 
move right backward, or both — based on what you're trying to find.

When to Use It
Two pointers work when:

* The array is sorted (or can be sorted)
* You're looking for a pair (or triplet) that satisfies a condition
* You need to partition or remove elements in-place
* You're comparing from both ends (palindrome check, container problem)


The 3 Flavors
1. Opposite Ends → Move Inward
Classic pattern. Start left at index 0, right at last index.
Problem: Two Sum (sorted array) — find pair that adds to target

[ 2, 7, 11, 15 ],  target = 9
  L            R

sum = 2 + 15 = 17  → too big  → move R left
sum = 2 + 11 = 13  → too big  → move R left
sum = 2 +  7 =  9  → FOUND ✅


def two_sum_sorted(nums, target):
    left, right = 0, len(nums) - 1
    
    while left < right:
        s = nums[left] + nums[right]
        if s == target:
            return [left, right]
        elif s < target:
            left += 1      # need bigger sum → move left forward
        else:
            right -= 1     # need smaller sum → move right back


2. Same Direction → Fast & Slow
Both pointers start at 0, but one runs ahead. Used for removing duplicates, 
filtering in-place.

Problem: Remove duplicates from sorted array

[ 1, 1, 2, 3, 3, 4 ]
  s
  f

slow = tracks where to write next valid element
fast = scans ahead looking for new values

def remove_duplicates(nums):
    slow = 0
    
    for fast in range(1, len(nums)):
        if nums[fast] != nums[slow]:   # found a new value
            slow += 1
            nums[slow] = nums[fast]    # write it at slow's position
    
    return slow + 1


3. Anchor + Runner (Sliding variation)
One pointer stays fixed as an anchor while the other explores. 
Common in 3Sum, container problems.

The Decision Rule (what to ask yourself)
At each step, ask:

  Is the current condition satisfied?
    → YES → record answer, move both
    → NO  → which pointer's value is "responsible"?
              → move THAT pointer


The Problem
You have an array of heights. Two lines form a container. 
Find the pair that holds the most water.

height = [ 1, 8, 6, 2, 5, 4, 8, 3, 7 ]
            0  1  2  3  4  5  6  7  8

Water = min(left_height, right_height) × distance between them
[ 1, 8, 6, 2, 5, 4, 8, 3, 7 ]
  L                          R

water = min(1, 7) × (8 - 0) = 1 × 8 = 8

Step 1
Is condition satisfied?  →  we want MAXIMUM water
Is this the max so far?  →  8  (record it)

Which pointer is responsible for the low water?
  left height  = 1  ← THIS is the bottleneck (min value)
  right height = 7

→ move LEFT

[ 1, 8, 6, 2, 5, 4, 8, 3, 7 ]
     L                      R

water = min(8, 7) × (8 - 1) = 7 × 7 = 49  ← new max


Step 2
Which pointer is responsible?
  left height  = 8
  right height = 7  ← THIS is the bottleneck

→ move RIGHT
[ 1, 8, 6, 2, 5, 4, 8, 3, 7 ]
     L                   R

water = min(8, 3) × (7 - 1) = 3 × 6 = 18

Step 3
Which pointer is responsible?
  left height  = 8
  right height = 3  ← bottleneck

→ move RIGHT
[ 1, 8, 6, 2, 5, 4, 8, 3, 7 ]
     L                R

water = min(8, 8) × (6 - 1) = 8 × 5 = 40
